## 一、绿盟升级描述的标准结构(用户必看)

绿盟所有产品线升级包的描述基本都用**【方括号段头】**做结构化分段。常见段头按出现频次:

| 段头 | 出现次数 | 含义 |
|---|---|---|
| 【升级说明】 | 81 | **最重要的"如何升级"段** — 含前置版本、是否重启、是否中断业务、预计耗时 |
| 【升级基础版本】 | 48 | 前置版本(支持从哪个版本开始升级)|
| 【升级版本】 | 47 | 同上,绿盟不同产品措辞略不同 |
| 【注意事项】 | **44** | **用户最该看的段** — 含备份、HA断开、md5校验、中断时长等 |
| 【系统升级】 | 32 | 系统升级路径说明 |
| 【前置版本】 | 14 | 同升级基础版本 |
| 【配套联动版本】 | 24 | 与其他产品联动所需版本 |
| 【升级步骤】 | 15 | 操作步骤 |
| 【HA】 | 13 | 高可用相关警示 |

**绿色区域**:有结构化段头的升级包(190 条,占 19.6%)— **用户应该优先按这些段头读**,信息最完整。
**黄色区域**:无段头但内容详尽的(规则库升级包、CVE 修复包)— 主要看 CVE 列表和"基于xx版本升级"措辞。
**红色区域**:只有一段话简述(系统类升级包部分)— 信息缺失,**建议升级前人工到绿盟下载页核对**。

## 二、绿盟"基于xx版本"措辞套路全集

绿盟描述里表达「升级前置条件」的措辞**至少有 6 种**,整理如下,后续订阅规则匹配或邮件正文渲染都能复用:

| 措辞 | 出现条数 | 含义 | 典型样例 |
|---|---|---|---|
| **仅支持 X 版本(及以上/以上)升级** | 248 | 前置版本 | "仅支持在固件版本 5.6R11F00 之上,引擎版本 5.6R11F00 及以上,5.6R11F01 以下升级" |
| **支持在 X 之上/上 升级** | 236 | 同上,措辞变体 | "支持在 V6.0.5.1.40649 及以上版本升级" |
| **仅支持在 X** | 122 | 强调支持范围 | "仅支持在 NX3/NX5 系列 HD 型号设备的升级" |
| **基于 X 版本升级** | 67 | 多见于 NISS/UTM 等系统升级包 | "升级包必须基于标准 NISS F02 当前版本 V6.0R00F02SP04-7603 的版本进行升级" |
| **X 及以上、X 以下(范围)** | 73 | 严格上下限 | "引擎版本 V2.0R00F05 及以上、V2.0R01F00 以下" |
| **不支持 X 升级** | 44 | **反向警示** | "不支持海光设备升级"、"32 位系统不能升级此升级包" |

**关键发现**:
- **反向警示(44 条)** 比**正向版本要求(73 条)** 还重要 — 因为管理员往往会忽略"我的设备是不是被排除在外"
- 「不支持/不能/勿」是绿盟硬红线,推送邮件应**显著标注**
- 同一产品(如 WAF V6.0.8)会有**多个 chain**:HD 标品、海光系列、飞腾系列 — 推送时要按用户设备 chain 精准匹配,否则会推错

## 三、各产品线重点关注事项(按用户最关心的排序)

### 1. WEB应用防护系统(WAF) ⭐⭐⭐⭐⭐ 最高优先级

WAF 是**多 chain 并列**(标品 + 海光 + 飞腾),用户最容易选错版本。**39 个升级包覆盖 3 个 series**。

**关键样例**(id=5418,WAF V6.0.8 系统升级包):
```
绿盟WAF NX3/NX5系列WAF V6.0.8.1系统升级文件,适用于NX3/NX5系列HD型号设备的升级,
不支持海光设备升级。
【前置版本】6081.69905以及以上
```

**用户必须确认 3 件事**:
1. **设备型号**:HD 标品 / 海光(HH)/ 飞腾 FT(NX3/NX5-HFA 或 NX5-HF) — **不能混装**
2. **前置版本**:从 `WAF V6.0.4 → V6.0.5 → V6.0.6 → V6.0.7 → V6.0.8` 是逐级升级链
3. **飞腾系列特殊**(id=5435):「**该版本不支持镜像模式**」

**核心注意事项(必看):**
- 不支持海光设备升级 → HD 系列升级包不能推到海光设备
- 不支持镜像模式 → 飞腾 HFA 系列部署模式限制
- 32 位系统不能升级 → 升级前确认 OS 位数
- 升级后需重启 → 业务中断窗口期必须规划

---

### 2. 安全认证网关(SAG) ⭐⭐⭐⭐

SAG 升级包的【注意事项】是**绿盟全产品里写得最规范的**,模板化:

**模板样例**(id=6038,SAG V2.0R02F03 系统升级包):
```
1. 支持V2.0R02F03 X86架构标品版本进行升级
2. 需搭配【2.301.x】版本的客户端使用,升级完成后请继续上传对应的客户端升级包
3. 本升级包升级后不会重启,升级完成后刷新页面重新登录即可,如登录后出现页面异常,
   请尝试清除浏览器缓存
4. 如超过30分钟升级进度条未结束,可能为令牌到期,请尝试刷新页面后重新登录
5. 升级前,请务必校验升级包的md5值,确认升级包的完整性
6. 不支持重复升级,升级前请备份配置,升级失败需初始化回退重新升级
7. 升级会中断业务,预计时间20-30分钟
```

**7 条通用注意事项**:
1. 支持的具体子版本号(X86/飞腾/海光三个架构分别有不同子版本)
2. **配套客户端版本号** — 必须同步升级客户端
3. 是否重启(本包:不重启) + 浏览器缓存清理
4. 升级超时阈值(30 分钟)
5. **MD5 校验**(绿盟反复强调)
6. **不支持重复升级** + 备份要求
7. **业务中断时长**(20-30 分钟)

**chain 维度**:SAG 有 X86 标品 / 海光 / 飞腾 / 客户端 4 个并列 chain,跨架构不能混装。

---

### 3. 网络入侵检测系统(IDS) / WEB应用漏洞扫描系统(WVSS) ⭐⭐⭐

**IDS 样例**(id=6234):
```
本升级包为入侵防护特征库升级包,仅支持在固件版本5.6R11F00之上,引擎版本
5.6R11F00及以上,5.6R11F01以下升级。升级包为全量升级包。升级后固件版本和
引擎版本不变,规则版本变为5.6.11.45151。
```

**规则库升级包通用模式**:
- **前置**:固件版本 + 引擎版本双重限制(范围式)
- **类型**:几乎都是**全量升级包**(620 条里 95% 规则库都是全量)
- **变更**:只改规则版本,**固件/引擎版本不变**
- **重启**:绝大多数**不需要**(规则库升级是热加载)

### 4. 绿盟综合威胁探针(UTS) ⭐⭐⭐⭐

**多 chain 复杂**(标准 + CW 定制 + 海光等):

**引擎升级包样例**(id=5669):
```
本升级包为绿盟综合威胁探针引擎升级包,支持在引擎版本V2.0R00F06SP04(build为50423)、
V2.0R00F06SP05(build为50500)、V2.0R00F06SP06(build为56824)、V2.0R00F06SP06H01
(build为56850)的标准版本(非定制版本)基础上进行升级。
```

**用户重点关注**:
- **build 号 + 版本号双重匹配** — 用户必须知道自己的精确 build 号
- **明确区分"标准版/非定制版"** — 定制版用户不能直接升
- 升级后固件版本不变,引擎版本变更

### 5. 智能安全运营平台 / 抗拒绝服务系统(ADS/NTA/ISOP) ⭐⭐⭐

**ADS 样例**(id=6048)用了**结构化英文章头**(与中文【】段头不同):
```
[Version No.] V4.5R90F04.sp06
[Source Version] V4.5R90F04, V4.5R90F04.sp01, ..., V4.5R90F04.sp05
[Target Version] V4.5R90F04.sp06
[Matching Versions of Collaborative Devices] NTA: V4.5R90F02, ...
```

**用户重点关注**:

- 子版本号逐级(V4.5R90F04 → V4.5R90F04.sp01 → ... → V4.5R90F04.sp06)

```
升级后出厂版本会保持电子盘所安装版本不变,当前版本信息变更为:V6.0R00F02SP05-8820,
```

## 五、按"用户最该关注的 7 件事"重新排序

如果只能给用户推一条邮件,内容应包含这 7 项:

| # | 关注点 | 关键句(优先级最高) |
|---|---|---|
| 1 | **我的设备能用吗?** | "**不支持XX设备/不支持XX模式/不支持32位**" — 出现就显著标注 |
| 2 | **基于哪个版本?** | "仅支持在 V6.0.X.X 及以上版本升级" — 写出精确版本号 |
| 3 | **要重启吗?** | 标品升级包通常不需要;系统/引擎升级包通常需要 |
| 4 | **会中断业务吗?** | SAG 写明 20-30 分钟;WAF 写明 60 秒 |
| 5 | **要不要备份?** | SAG/通用补丁:需要;规则库:不需要 |
| 6 | **联动设备要配套升吗?** | ISOP/NISS 必须配 NF/WAF/LAS/OSMS |
| 7 | **客户端要不要升?** | SAG 强制要求客户端版本配套 |

## 六、给监控系统的功能建议(基于以上数据)

如果用户希望把这个整理**落地到系统里**,有三件事可以做:

### 6.1 description_raw 结构化解析(本次未实施)

`description_raw` 当前只有邮件正文渲染用,**没结构化字段**。建议:
- 新增 `description_structured` JSON 字段,解析出 `engine_min / engine_max / firmware_min / pkg_incremental_or_full / restart / interrupt_minutes / backup_required / linked_components / device_restrictions`
- 历史 968 条回填一次即可
- 后续订阅规则 / 仪表盘 / 邮件都能复用

### 6.2 推送正文加"重点关注"红色块(本次未实施)

邮件正文现在只显示 desc + 按钮。建议在 desc 前插一段:
```html
<div style="background:#fff3cd;border-left:4px solid #ff9800;padding:12px">
  <strong>⚠️ 重点关注</strong>
  <ul>
    <li>前置版本:V6.0.5.1.40649 及以上</li>
    <li>升级后需重启</li>
    <li>预计耗时:60 秒</li>
    <li>不支持 32 位系统</li>
  </ul>
</div>
```
数据来源就是 6.1 解析的 JSON。

### 6.3 仪表盘"注意事项"标签(本次未实施)

左侧产品卡片加红色"⚠️"角标,**仅当**该产品最新升级包含「不支持/不能/勿」类反向警示时显示。点击角标展开注意事项全文。
