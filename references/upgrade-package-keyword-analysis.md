# 升级包描述关键词分析报告

> 版本：v1.0
> 日期：2026-05-28
> 数据来源：snapshots.description_raw（709条记录）

---

## 1. 分析概述

- **分析目的**：从升级包描述文本中提取关键操作提示，归纳高亮关键词
- **分析范围**：709条 `description_raw` 字段（distinct）
- **分析工具**：Python + SQLite + 正则表达式
- **分析结果**：提炼出 P0/P1/P2 三层高亮关键词体系

## 2. 各分类命中统计

### 2.1 【前置版本】相关（影响升级是否成功）

| 表达模式 | 命中数 | 占比 |
|---------|-------|------|
| `系统版本为XX及以上` | 40 | 5.6% |
| `请在XX版本及以上版本进行升级` | 10 | 1.4% |
| `升级前请确保当前系统版本` | 6 | 0.8% |
| `仅支持XX及以上版本` | 3 | 0.4% |
| `需在XX引擎版本上升级使用` | 2 | 0.3% |
| `【前置版本】` + 版本号 | 14 | 2.0% |
| `配合前置升级包使用` | 1 | 0.1% |

**关键发现**：
- 版本约束表达方式分散，无统一格式，需多模式组合匹配
- `【前置版本】` 标签是最明确的版本约束标识
- 部分描述用"请在XX版本以上/及以上版本"表述，模式不统一

### 2.2 【重启设备】相关

| 表达模式 | 命中数 |
|---------|-------|
| `重启设备` | 59 |
| `需要重启设备` | 10 |
| `无需重启设备和引擎` | 9 |
| `升级后自动重启设备` | 2 |
| `重启后生效` | 1 |

**典型上下文**：
- `升级完成后，需要重启设备以使设备后续正常运作`
- `升级后需重启设备以使升级包生效`
- `升级后无需重启设备和引擎` ✅
- `本升级包升级后会自动重启设备生效`
- `请不要强制硬重启设备` ⚠️

### 2.3 【重启引擎】相关

| 表达模式 | 命中数 |
|---------|-------|
| `重启引擎` | 63 |
| `引擎重启` | 5 |
| `升级后自动重启引擎` | 53 |
| `无需重启设备和引擎` | 9 |

**关键发现**：
- "升级后自动重启引擎"是最高频的重启类描述（53条）
- 与"重启设备"独立描述，升级包可能只重启引擎不重启设备，或只重启设备不重启引擎
- 引擎重启通常会导致流量/会话中断（`会话中断`出现频率 147条）

### 2.4 【业务影响】相关

| 表达模式 | 命中数 | 说明 |
|---------|-------|------|
| `会话中断` / `流量中断` | ~147条 | 引擎重启导致 |
| `网络中断` | ~10条 | 设备重启/系统重启导致 |
| `WEB短时无法访问` | 多次 | 升级过程中WEB不可用 |
| `业务空闲` | ~147条 | 升级时机提醒 |

**典型耗时**：
- `耗时约2~5分钟` — 引擎重启类
- `耗时约3~10分钟` — 设备重启类（自动）
- `耗时约5~10分钟` — 设备重启类
- `耗时约8~10分钟` — 重启转发引擎类
- `耗时约15~30分钟` — 整机重启类（影响最大）

### 2.5 【型号/设备限制】相关

| 限制类型 | 示例 |
|---------|------|
| `不支持海光设备` | NX3/NX5 HH海光型号专用包 |
| `不支持HD型号` | NX3/NX5 HD型号不可用 |
| `仅支持NX3/NX5 HD系列` | 特定型号限制 |
| `不支持软件化部署` | 硬件/虚拟化专用 |
| `不支持HD2100C/HDB2105...` | 具体型号黑名单 |
| `需要串口升级` | 不支持WEB导入 |

### 2.6 【全量 vs 增量】升级包类型

| 类型 | 命中数 | 特征 |
|------|-------|------|
| `全量升级包` | ~373条 | 可跨多版本升级，覆盖合并包 |
| `增量升级包` | ~76条 | 需逐级升级 |

**关键发现**：全量升级包描述中常包含"固件版本不变，引擎版本变更为XXX"或"固件版本和引擎版本不变"。

## 3. 特殊段落格式分析

### 3.1 【升级建议】段落（规则包专属）

出现在规则库升级包描述中，格式高度统一：

```
五、升级建议
(1)请在64位系统版本V6.0R09F00.29386582及以上版本进行升级。
(2)如果设备版本大于规则包版本，即包含此规则包所有规则。
(3)升级后无需重启设备和引擎。
(4)如果要应用规则，请为防护站点勾选对应的规则。
(5)如系统正在初始化(如：系统开机，系统升级，系统还原)，请等待初始化完毕再进行规则升级操作。
(6)如遇到界面显示规则包升级成功，但是没有加载到新的规则的问题，可尝试先进行规则还原，然后再次进行规则升级
```

此段落是规则包特有的"五、升级建议"，可作为规则包的独立高亮区块。

### 3.2 【升级说明】段落（系统包专属）

出现在系统升级包描述中，格式不统一但内容相似：

```
【升级说明】
1. 升级前请确保当前系统版本等于或高于V6.0.4.0.26919
2. 支持从6.0.4.0.26919、6.0.4.0.27063...版本升级
3. 升级后需重启系统以使升级包生效
```

## 5. 分析代码

```python
import sqlite3, re
from collections import Counter

db_path = '/root/nsfocus-monitor/data/nsfocus_monitor.db'
conn = sqlite3.connect(db_path)
conn.text_factory = str
cur = conn.cursor()

cur.execute("""
    SELECT description_raw
    FROM snapshots
    WHERE description_raw IS NOT NULL AND description_raw != ''
""")
rows = [r[0] for r in cur.fetchall()]
# 总样本数: 709
```
