# 需求规格说明

## 1. 项目背景

绿盟科技通过升级站点 update.nsfocus.com 定期发布安全产品升级包。目前绿盟不会主动通知客户升级包发布，客户需自行登录站点查看，导致：
- 关键安全更新可能被遗漏
- 客户无法及时了解规则库/系统升级包发布
- 运维人员需要手动逐个检查多个产品

本平台解决：自动监控 → 智能检测 → 多渠道通知 → 客户管理闭环。

## 2. 用户角色

| 角色 | 说明 |
|------|------|
| **系统管理员** | 管理平台全局配置、管理所有客户、推送通知 |
| **普通运营人员** | 管理自己的 Session、订阅规则、查看推送历史 |

一期不做角色权限细分，所有用户平权。

## 3. 功能需求

### 3.1 Session 管理

- **FR-01**: 用户可手动输入绿盟站点的 PHPSESSID
- **FR-02**: 系统展示 Session 状态（有效/过期/未知）
- **FR-03**: Session 过期前主动发送告警（企微/钉钉/飞书通知管理员）
- **FR-04**: 多用户 Session 池化，一个失效自动切换其他用户的可用 Session
- **FR-05**: Session 加密存储（AES）

### 3.2 内容源采集

- **FR-06**: Phase 1 支持 6 个产品: WAF / IPS / IDS / RSAS / NF / UTS
- **FR-07**: 自动遍历三级页面结构（产品页 → 版本页 → 包列表页）
- **FR-08**: 解析升级包字段：文件名、版本号、MD5、大小、描述、发布时间、下载 ID
- **FR-09**: 从描述中提取结构化信息：新增/修改/删除规则、前置版本、紧急程度
- **FR-10**: 请求间随机延迟 1-3 秒，防止限流
- **FR-11**: 使用 HEAD 请求检查 Last-Modified，无变化则跳过详情抓取
- **FR-12**: 解析器健康检查（解析到的包数异常变化时告警）

### 3.3 变化检测

- **FR-13**: 新增检测：本次有、上次无的包标记为新增
- **FR-14**: 回退检测：上次有、本次连续 N 次（默认 2 次）无的标记为回退
- **FR-15**: 防误报：单次缺失不判定回退，需连续确认
- **FR-16**: 回退通知标记为紧急，内容突出"⚠️ 软件包已撤回"

### 3.4 客户管理

- **FR-17**: 客户档案：名称、公司、联系人、持有产品列表、持有版本
- **FR-18**: 客户与通知渠道关联
- **FR-19**: 支持按客户持有的产品/版本自动过滤通知

### 3.5 订阅规则

- **FR-20**: 规则包含：匹配条件（产品/版本/包类型）、通知渠道、推送策略
- **FR-21**: 推送延迟：可设置 0~30 天延迟
- **FR-22**: 延迟策略三种：重置（新包重置计时器）、追加（各自计时）、截止窗口（窗口内合并）
- **FR-23**: 最小通知间隔：同产品同渠道 N 天内最多推送 M 次
- **FR-24**: 静默时段：可配置不推送的时间段（如 22:00~08:00）
- **FR-25**: 回退通知不受延迟/静默限制，即时推送

### 3.6 通知渠道

- **FR-26**: 企业微信机器人（Webhook + Markdown）
- **FR-27**: 钉钉机器人（Webhook + Markdown）
- **FR-28**: 飞书机器人（Webhook + 富文本）
- **FR-29**: 邮件（SMTP，支持附件）
- **FR-30**: 邮件附件策略：规则包 ≤10MB 自动下载并附件发送；系统包 >10MB 仅提供链接
- **FR-31**: 通知模板支持变量：产品、版本、包类型、文件名、MD5、大小、发布时间、新增/修改规则摘要、依赖版本、下载链接、紧急程度

### 3.7 推送确认

- **FR-32**: 每次推送完成后，向对应 IM 群发送送达确认消息
- **FR-33**: 确认消息列出：推送内容摘要、各渠道送达状态（成功/失败）、时间戳

### 3.8 推送历史

- **FR-34**: 记录所有推送历史，支持按时间/产品/客户筛选
- **FR-35**: 支持从历史记录一键重新推送
- **FR-36**: 展示推送统计：本月推送次数、各产品发布频次

### 3.9 Web 管理面板

- **FR-37**: 用户注册/登录（密码 bcrypt 哈希）
- **FR-38**: 首次使用引导流程：① 获取 Session → ② 添加渠道 → ③ 创建规则
- **FR-39**: 内容预览：配置监控项后可手动触发一次抓取预览结果
- **FR-40**: 手动推送：可即时推送一条自定义通知（不依赖自动检测）
- **FR-41**: 监控仪表盘：Session 状态、最近抓取时间、各产品包数、推送统计

### 3.10 安全

- **FR-42**: 敏感字段 AES 加密存储（Session、SMTP 密码、Webhook URL）
- **FR-43**: Web 面板 JWT 认证
- **FR-44**: 操作审计日志：登录、配置变更、手动推送

### 3.11 运维

- **FR-45**: 结构化日志（DEBUG/INFO/WARN/ERROR）
- **FR-46**: systemd 服务管理
- **FR-47**: 采集失败自动告警（独立于抓取链路）

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 全量检查 6 个产品 ≤ 3 分钟 |
| 内存 | 运行时 ≤ 300MB（含 Flask + SQLite） |
| 可靠性 | 采集失败自动重试 3 次 |
| 可扩展 | 新增产品只需添加配置文件，无需修改核心逻辑 |
| 数据保留 | 快照数据保留 180 天，推送历史保留 365 天 |

## 5. Phase 2 预留

- 博客/公众号 RSS 监控 (RssCollector)
- 微信公众号内容监控 (WechatMpCollector)
- 通知模板可视化编辑器
- 推送效果度量（打开率、点击率）
- 季度报告导出 (PDF/Excel)
- 客户自助订阅门户（客户登录选择订阅内容）
