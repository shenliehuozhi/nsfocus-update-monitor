# 部署运维

## 环境要求

- OS: CentOS 7+ / Ubuntu 18+ / macOS
- Python: 3.8+
- 内存: ≥ 400MB可用
- 磁盘: ≥ 500MB可用空间

## 一键部署（推荐）

```bash
git clone https://github.com/shenliehuozhi/nsfocus-update-monitor.git
cd nsfocus-monitor
bash start.sh
```

**`start.sh` 自动完成**：

## SSH 隧道远程访问（推荐）

本地默认监听 `127.0.0.1:9999`，远程访问通过 SSH 隧道，安全且无需暴露端口。

### Linux / macOS

```bash
# 本地端口转发：远程服务器的 9999 → 本地 9999
ssh -L 9999:127.0.0.1:9999 user@your-server.com -N

# 浏览器直接访问
http://127.0.0.1:9999
```

### Windows（PowerShell / WSL）

```powershell
# PowerShell
ssh -L 9999:127.0.0.1:9999 user@your-server.com -N
```

### Windows（PuTTY）

```
Connection → SSH → Tunnels
Source port: 9999
Destination: 127.0.0.1:9999
Type: Local
```

> 所有 API 请求和页面访问均通过加密的 SSH 隧道传输，适合内网/服务器不可直接暴露端口的场景。

### 自定义监听地址

```bash
# .env 中修改（默认 127.0.0.1，仅监听本地）
MONITOR_HOST=0.0.0.0    # 改为全局监听（需配合防火墙或 VPN）
MONITOR_PORT=9999
```

## systemd 服务（生产环境）

> 生产环境部署建议配合 systemd 管理服务进程，并通过 SSH 隧道访问 Web 界面。

```bash
# 创建环境变量文件
cat > /root/nsfocus-monitor/.env << 'EOF'
MONITOR_SECRET_KEY=$(python3 -c "import secrets; print(secrets.token_hex(32))")
MONITOR_JWT_SECRET=$(python3 -c "import secrets; print(secrets.token_hex(32))")
MONITOR_PORT=9999
MONITOR_DATA_DIR=/root/nsfocus-monitor/data
MONITOR_LOG_DIR=/root/nsfocus-monitor/logs
MONITOR_COLLECT_INTERVAL=4
MONITOR_ROLLBACK_CONFIRM=2
MONITOR_ATTACHMENT_MAX_SIZE=10485760
EOF

# 安装服务
cat > /etc/systemd/system/nsfocus-monitor.service << 'EOF'
[Unit]
Description=NSFOCUS Update Monitor
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/nsfocus-monitor
EnvironmentFile=/root/nsfocus-monitor/.env
ExecStart=/root/nsfocus-monitor/venv/bin/python /root/nsfocus-monitor/run.py
Restart=on-failure
RestartSec=30
StartLimitBurst=5
StartLimitIntervalSec=60
MemoryMax=400M
StandardOutput=append:/root/nsfocus-monitor/logs/service_stdout.log
StandardError=append:/root/nsfocus-monitor/logs/service_stderr.log

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable nsfocus-monitor
systemctl start nsfocus-monitor
systemctl status nsfocus-monitor
```

### 关键日志关键词

| 关键词 | 含义 | 处理 |
|--------|------|------|
| `WARNING - Session xxx expired` | Session 过期 | 提醒用户更新 PHPSESSID |
| `ERROR - Collection failed` | 采集失败 | 检查绿盟站点可访问性 |
| `WARNING - Health check degraded` | 解析器异常 | 检查页面是否改版 |
| `ERROR - All sessions exhausted` | 无可用 Session | 紧急：所有用户更新 Session |
| `INFO - New package detected` | 正常：发现新包 | 无需处理 |

## 故障处理

### 采集异常诊断三步法

> 适用场景：推送内容为空、描述字段缺失、包信息不完整

**第一步：curl 验证源站**
```bash
# 用当前最新 Session 访问列表页，看返回内容
curl -b "PHPSESSID=$(sqlite3 /root/nsfocus-monitor/data/nsfocus_monitor.db 'SELECT value FROM settings WHERE key=\"sessions\"' | python3 -c 'import sys,json; s=json.load(sys.stdin); print(s[0][\"value\"] if s else "")')" \
  https://update.nsfocus.com/update/wafIndex -s | head -50

# 访问详情页验证完整字段
curl -b "PHPSESSID=<session>" https://update.nsfocus.com/update/wafDetail?id=xxx -s | python3 -c "import sys,json; d=json.load(sys.stdin); print(list(d.keys())); print(d.get('description',''))"
```

**第二步：控制台检查采集日志**
```bash
# 找采集入口的第一条日志
grep "INFO - Collection started\|Collection completed\|Collecting" /root/nsfocus-monitor/logs/app.log | tail -20

# 找异常关键字
grep -E "ERROR|WARNING|failed|null|empty" /root/nsfocus-monitor/logs/app.log | tail -30
```

**第三步：代码审查字段映射**
```bash
# 查后端解析逻辑
grep -n "description\|file_name\|md5\|file_size\|published" /root/nsfocus-monitor/routes/parsers.py
```

### 采集异常快速恢复

| 异常类型 | 表现 | 快速恢复 |
|----------|------|----------|
| Session 失效 | 采集日志 `All sessions exhausted` | 更新有效 PHPSESSID |
| 页面改版 | 解析字段为空 / 页面结构变化 | 检查 parsers.py，定位字段映射 |
| 调度器卡住 | 上次采集时间长期未更新 | `curl -X POST http://localhost:9999/api/system/collect -d '{"mode":"quick"}'` 手动触发 |
| 数据不完整 | 部分字段空（description/md5等） | 等下一次 Full 扫描自动补全，或手动触发 full 扫描 |

### Session 失效恢复
```bash
# 查看所有 Session 状态
curl -s http://localhost:9999/api/sessions

# 添加新 Session（平台 UI 操作或 API）
# 新 Session 添加后自动参与采集，无需重启服务
```

### 紧急：所有 Session 失效
1. 登录 update.nsfocus.com 获取新 PHPSESSID
2. 在平台 Session 管理页面添加新 Session
3. 验证有效性（平台会显示验证结果）
4. 触发一次 Quick 采集确认恢复
```bash
curl -X POST http://localhost:9999/api/system/collect \
  -H "Content-Type: application/json" \
  -d '{"mode":"quick"}'
```

### 推送内容为空 / 关键字段缺失
1. **诊断**：查看 app.log 中该包的采集时间戳和解析结果
2. **验证**：用当时有效的 Session 手动访问详情页，确认字段存在
3. **修复**：如果是页面改版，更新 parsers.py 对应解析器
4. **补全**：触发 Full 扫描重新采集
```bash
curl -X POST http://localhost:9999/api/system/collect \
  -H "Content-Type: application/json" \
  -d '{"mode":"full"}'
```

### 关键日志关键词（补充）

| 关键词 | 含义 | 处理 |
|--------|------|------|
| `WARNING - 采集异常.*关键字段` | 单个包关键字段缺失 | 正常：下次 Full 扫描自动补全 |
| `ERROR - 采集失败\|Connection` | 网络或绿盟站点异常 | 检查网络和绿盟站点可访问性 |
| `WARNING - Session.*expired` | Session 过期 | 刷新 PHPSESSID |
| `ERROR - All sessions exhausted` | 无可用 Session | 紧急：所有 Session 均失效，需重新添加 |

### 常见故障（原有）

| 问题 | 排查步骤 |
|------|----------|
| 无法启动 | `journalctl -u nsfocus-monitor -n 50` 查看错误 |
| 内存超限 | 检查 `MemoryMax` 是否生效，调整附件下载策略 |
| 采集失败 | `curl -b "PHPSESSID=xxx" https://update.nsfocus.com/update/wafIndex` 手动测试 |
| 通知不发 | 检查订阅规则是否启用、延迟队列状态、渠道配置 |
| 推送被频率限制 | 查看 `GET /api/system/rate-limits`，用 `POST /api/system/rate-limits/reset` 重置 |
| 数据库锁定 | SQLite 单写，确保没有多个进程同时写入 |

### 频率限制管理

```bash
# 查看当前封禁状态
curl -u admin:<password> http://127.0.0.1:9999/api/system/rate-limits

# 重置特定 key
curl -u admin:<password> -X POST http://127.0.0.1:9999/api/system/rate-limits/reset \
  -H 'Content-Type: application/json' -d '{"key":"user@example.com"}'

# 重置全部（不传 key）
curl -u admin:<password> -X POST http://127.0.0.1:9999/api/system/rate-limits/reset \
  -H 'Content-Type: application/json' -d '{}'
```

## 安全加固建议

默认仅监听 `127.0.0.1`，本地访问无需额外配置。如需全局监听请在 `.env` 中修改 `MONITOR_HOST=0.0.0.0` 并配置防火墙或 VPN。

```bash
# 全局监听模式（需额外防护）
MONITOR_HOST=0.0.0.0
```
