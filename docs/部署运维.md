# 部署运维

## 环境要求

- OS: CentOS 7+ / Ubuntu 18+ / macOS
- Python: 3.8+
- 内存: ≥ 400MB可用
- 磁盘: ≥ 500MB可用空间

## 一键部署（推荐）

```bash
git clone https://github.com/shenliehuozhi/nsfocus-update-monitor.git
cd nsfocus-monitor
bash start.sh
```

## SSH 隧道远程访问（推荐）

本地默认监听 `127.0.0.1:9999`，远程访问通过 SSH 隧道，安全且无需暴露端口。

### Linux / macOS

```bash
# 本地端口转发：远程服务器的 9999 → 本地 9999
ssh -L 9999:127.0.0.1:9999 user@your-server.com -N

# 浏览器直接访问
http://127.0.0.1:9999
```

### Windows（PowerShell / WSL）

```powershell
# PowerShell
ssh -L 9999:127.0.0.1:9999 user@your-server.com -N
```

### Windows（PuTTY）

```
Connection → SSH → Tunnels
Source port: 9999
Destination: 127.0.0.1:9999
Type: Local
```

> 所有 API 请求和页面访问均通过加密的 SSH 隧道传输，适合内网/服务器不可直接暴露端口的场景。

### 自定义监听地址

```bash
# .env 中修改（默认 127.0.0.1，仅监听本地）
MONITOR_HOST=0.0.0.0    # 改为全局监听（需配合防火墙或 VPN）
MONITOR_PORT=9999
```

## systemd 服务（生产环境）

> 生产环境部署建议配合 systemd 管理服务进程，并通过 SSH 隧道访问 Web 界面。

```bash
# 创建环境变量文件
cat > /root/nsfocus-monitor/.env << 'EOF'
MONITOR_SECRET_KEY=$(python3 -c "import secrets; print(secrets.token_hex(32))")
MONITOR_JWT_SECRET=$(python3 -c "import secrets; print(secrets.token_hex(32))")
MONITOR_PORT=9999
MONITOR_DATA_DIR=/root/nsfocus-monitor/data
MONITOR_LOG_DIR=/root/nsfocus-monitor/logs
MONITOR_COLLECT_INTERVAL=4
MONITOR_ROLLBACK_CONFIRM=2
MONITOR_ATTACHMENT_MAX_SIZE=10485760
EOF

# 安装服务
cat > /etc/systemd/system/nsfocus-monitor.service << 'EOF'
[Unit]
Description=NSFOCUS Update Monitor
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/nsfocus-monitor
EnvironmentFile=/root/nsfocus-monitor/.env
ExecStart=/root/nsfocus-monitor/venv/bin/python /root/nsfocus-monitor/run.py
Restart=on-failure
RestartSec=30
StartLimitBurst=5
StartLimitIntervalSec=60
MemoryMax=400M
StandardOutput=append:/root/nsfocus-monitor/logs/service_stdout.log
StandardError=append:/root/nsfocus-monitor/logs/service_stderr.log

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable nsfocus-monitor
systemctl start nsfocus-monitor
systemctl status nsfocus-monitor
```

### 关键日志关键词

| 关键词 | 含义 | 处理 |
|--------|------|------|
| `WARNING - Session xxx expired` | Session 过期 | 提醒用户更新 PHPSESSID |
| `ERROR - Collection failed` | 采集失败 | 检查绿盟站点可访问性 |
| `WARNING - Health check degraded` | 解析器异常 | 检查页面是否改版 |
| `ERROR - All sessions exhausted` | 无可用 Session | 紧急：所有用户更新 Session |
| `INFO - New package detected` | 正常：发现新包 | 无需处理 |

## 故障处理

### 采集异常诊断三步法

### 采集异常诊断三步法

> 适用场景：推送内容为空、描述字段缺失、包信息不完整

**第一步：确认采集是否真的跑过**
```bash
# 看最近一次采集的开始/结束时间
grep -E "Collection started|Collection completed|增量|Quick" /root/nsfocus-monitor/logs/app.log | tail -5

# 看调度器状态（是否卡住）
sqlite3 /root/nsfocus-monitor/data/nsfocus_monitor.db "SELECT key,value FROM system_settings WHERE key IN ('last_collect_at','collection_running','scheduler_enabled')"
```

**第二步：curl 验证源站**
```bash
# 用当前最新 Session 访问列表页（替换 PHPSESSID）
curl -s -b "PHPSESSID=<your_session>" https://update.nsfocus.com/update/wafIndex | head -50

# 验证详情页字段是否完整
curl -s -b "PHPSESSID=<your_session>" https://update.nsfocus.com/update/wafDetail?id=<id> | python3 -c "
import sys, json
d = json.load(sys.stdin)
print('description:', d.get('description','为空'))
print('file_name:', d.get('file_name','为空'))
print('md5_hash:', d.get('md5_hash','为空'))
print('file_size:', d.get('file_size','为空'))
"
```

**第三步：控制台日志 + 代码定位**
```bash
# 找该包的采集时间和错误信息
grep -E "关键字段|ERROR|WARNING" /root/nsfocus-monitor/logs/app.log | grep "<包名或产品名>" | tail -10

# 查解析器字段映射（description/file_name/md5_hash/file_size/published_at）
grep -n "description\|file_name\|md5_hash\|file_size\|published" /root/nsfocus-monitor/src/collectors/nsfocus.py | grep -v "^#\|# " | tail -20

# 查最近更新记录（看是否绿盟改版）
git -C /root/nsfocus-monitor log --oneline -10 -- src/collectors/nsfocus.py
```

### Session 失效恢复

**查看所有 Session 状态**
```bash
# API 查看（需 Bearer token，登录后获取）
curl -s -H "Authorization: Bearer <token>" http://127.0.0.1:9999/api/sessions | python3 -m json.tool

# 或数据库直接查
sqlite3 /root/nsfocus-monitor/data/nsfocus_monitor.db "SELECT id, name, status, heartbeat_status, last_heartbeat_at FROM user_sessions"
```

**添加新 Session**
1. 浏览器登录 https://update.nsfocus.com，手动复制 PHPSESSID Cookie
2. 平台 UI → Session 管理 → 添加 → 粘贴 → 验证
3. 新 Session 验证通过后自动参与采集，无需重启服务

### 紧急：所有 Session 失效（最高优先级）

**触发条件**：`ERROR - All sessions exhausted` 日志出现

**恢复步骤**：
1. 浏览器登录 https://update.nsfocus.com，用正常账号获取新 PHPSESSID（需确保登录过程完整，不要只复制残留 Cookie）
2. 在平台 Session 管理页面添加新 Session，验证通过
3. 确认 Session 状态为"活跃"（heartbeat_status = 正常）
4. 触发一次 Quick 采集验证：
```bash
curl -X POST http://127.0.0.1:9999/api/system/collect \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"mode":"quick"}'
```
5. 观察 app.log 确认采集正常启动，无 `All sessions exhausted` 重现

### 推送内容为空 / 关键字段缺失

**诊断路径**：
1. 查 app.log：该包的采集时间戳、解析结果、关键字段值
2. 用当时有效的 Session 手动 curl 详情页，确认绿盟页面该字段是否真实存在
3. 若字段在绿盟页面存在但采集为空 → parsers.py 字段映射错误
4. 若字段在绿盟页面也不存在 → 绿盟数据本身缺失，非本系统问题

**修复后补全**：
```bash
# 当前 Quick 采集覆盖所有活跃产品，字段完整的包自动更新
# 无需手动重采，下一次调度周期自然补全
```

### 常见故障对照表

| 问题 | 排查命令 | 快速恢复 |
|------|----------|----------|
| 服务无法启动 | `journalctl -u nsfocus-monitor -n 50` | 检查端口冲突、依赖缺失 |
| 采集一直失败 | `grep ERROR /root/nsfocus-monitor/logs/app.log \| tail -10` | 确认网络/Session |
| 推送没收到 | `sqlite3 /root/nsfocus-monitor/data/nsfocus_monitor.db "SELECT * FROM delivery_log ORDER BY sent_at DESC LIMIT 5"` | 检查订阅规则、渠道配置 |
| 队列积压 | `sqlite3 /root/nsfocus-monitor/data/nsfocus_monitor.db "SELECT COUNT(*) FROM delayed_queue WHERE status='pending'"` | 手动放行或等重试 |
| 频率限制 | `curl -s -u admin:<pwd> http://127.0.0.1:9999/api/system/rate-limits` | 等待自动解封或 `POST /api/system/rate-limits/reset` |
| 数据库锁定 | `sqlite3 /root/nsfocus-monitor/data/nsfocus_monitor.db "PRAGMA integrity_check"` | 检查是否有并发写进程 |
| 服务无响应（写锁阻塞） | `grep "execute slow\|database is locked" /root/nsfocus-monitor/logs/app.log`；`ps aux | grep run.py` 确认进程存活；`curl -w "%%{http_code}" http://localhost:9999/api/dashboard` 测试响应时间 | 重启服务：`pkill -f "python3.10 -B run.py"; sleep 2; /usr/bin/python3.10 -B run.py &` |
| 磁盘空间不足 | `df -h /root` | 清理旧日志、备份文件 |
