## 3. 核心设计决策

### 3.1 采集统一抽象

```
BaseCollector
  ├── collect()          → List[UnifiedContentItem]
  ├── check_health()     → CollectorHealth
  └── source_type        → str

NsfocusCollector(BaseCollector)
  ├── PRODUCTS 字典驱动（§3.6），可扩展至任意数量产品
  ├── 标准型：3级遍历（版本→包类型→table）
  ├── 递归型：最多4层深（遇table停止）
  ├── HTML 表格解析
  ├── Quick 模式：已知 URL hash 对比，变化才全量解析
  └── 描述结构化提取

RssCollector(BaseCollector)       # Phase 2
WechatMpCollector(BaseCollector)  # Phase 2
```

### 3.2 通知统一格式

```python
@dataclass
class NotificationMessage:
    title: str           # "WAF V6.0.9 规则包更新"
    summary: str         # 单行摘要
    fields: dict         # 结构化字段（详情）
    urgency: str         # normal | high | critical
    source_url: str      # 绿盟源页面链接
    download_url: str    # 下载直链
    attachment: Optional[bytes]  # 附件内容（≤10MB）
    attachment_name: str
```

每个 Notifier 实现 `send(message) → DeliveryResult`，返回送达状态。

### 3.3 推送确认链路

```
NotificationRouter.send()
  → for each channel in matched_channels:
      result = channel.notifier.send(message)
      # 记录发送结果
      save_delivery_log(message, channel, result)
  
  → # 对 IM 渠道发确认消息
  → for im_channel in im_channels_used:
      im_channel.notifier.send_confirmation(message, results)
```

### 3.4 Session 冗余策略

```
SessionPool:
  ├── 收集所有用户的活跃 PHPSESSID
  ├── 按优先级排序（最近验证有效 > 从未验证 > 最近失败）
  ├── 采集时取第一个有效 Session
  └── 采集失败 → 取下一个 Session 重试
```

### 3.5 延迟推送状态机

```
                ┌──────────┐

                └────┬─────┘
```

### 3.6 产品管理 — 自动发现 + 开关管理

> **设计原则**：用户无需改代码。产品列表由数据库驱动，通过 Web UI 管理。

#### 当前状态

硬编码 6 产品（`nsfocus.py` PRODUCTS 字典）。计划改造为 DB 驱动 + Web UI。

#### 自动发现机制

绿盟升级站点客户支持页面「软件升级」栏目列出了所有可升级产品。分析确认（2026-05-13）：
- **79 个产品链接**，全部遵循统一格式 `https://update.nsfocus.com/update/[xxx]`
- **无需 session**，公共页面直接可解析
- 过滤规则极简：`href 含 '/update/' 且不含 '/portal/'`（该页面本身就是专用产品列表，无噪音链接）

URL 规律统计：

| 模式 | 数量 | 示例 |
|------|------|------|
| `/update/xxxIndex` | ~38 | wafIndex, bmgIndex, adsIndex |
| `/update/listXxx` | ~13 | listIds, listIps, listScm |
| `/update/listXxxIndex` | ~8 | listDmsIndex, listSasICSIndex |
| `/update/XxxIndex`（大写开头） | ~15 | DsitIndex, AdbosIndex, CSSIndex |
| 例外 | ~5 | bsaUtsIndex, listAuroraIndex, websafeIndex |

特殊案例（无标准缩略名）：
- `ROS系统` → `/update/websafeIndex`
- `工程工具集` → `/update/apolloIndex`  
- `公共互联网安全（TDC）` → `/update/tdcIndex`

#### 产品生命周期

```
自动发现（访问绿盟页面）
  │
  ├─ 新发现产品 → is_active=0（默认禁用）
  │   └─ 用户手动打开开关 → is_active=1 → 开始采集
  │
  ├─ 已有产品 → 保持原状态
  │   ├─ is_active=1 → 正常采集
  │   └─ is_active=0 → 跳过不采集（历史快照保留）
  │
  └─ 不允许删除 → 历史快照数据有价值
```

#### 策略检测

首次导入默认 `strategy='standard'`。提供「批量检测」功能：有 session 时逐个分析产品首页结构（标准型/递归型），自动更正策略字段。用户也可手动修改。

#### 手动注册（兜底）

如自动发现遗漏产品，用户可在 Web UI 手动注册：
- 填写产品名称 + 入口 URL
- 点「🔍 检测」按钮 → 后端访问页面分析结构 → 自动推荐策略
- 用户可覆盖策略选择
- 保存后立即生效

#### 采集侧改动

```python
# 旧：硬编码
PRODUCTS = {'WAF': '/update/wafIndex', ...}
for name in ('RSAS', 'NF'):
    _collect_recursive(...)

# 新：DB 动态读取
def _get_products() -> list:
    """返回 is_active=1 的产品列表"""
    return query("SELECT id, name, entry_url, strategy FROM content_sources "
                 "WHERE source_type='nsfocus' AND is_active=1 ORDER BY name")

for src in _get_products():
    if src['strategy'] == 'recursive':
        _collect_recursive(src['id'], src['name'], src['entry_url'])
    else:
        _collect_standard(src['id'], src['name'], src['entry_url'])
```

#### DB 迁移

`content_sources` 表新增两列：
```sql
ALTER TABLE content_sources ADD COLUMN entry_url TEXT DEFAULT '';
ALTER TABLE content_sources ADD COLUMN strategy TEXT DEFAULT 'standard';
```

现有 6 产品回填 entry_url + strategy，保持 is_active=1。

### 3.7 扩展新产品（现方案）

#### 方式一：自动扫描（推荐）

系统设置页点击「🔄 重新扫描」→ 后端访问绿盟升级站点 → 发现全部 79 个产品 → 新发现的默认禁用 → 用户按需启用。

#### 方式二：手动注册（兜底）

系统设置页「+ 手动注册产品」弹窗 → 填写名称、URL、策略 → 保存。

#### 3.7.1 判断采集策略

先按标准型加入，跑 full 扫描。观察日志：
- `产品名: N items`（N > 0）→ 标准型正确
- `产品名: 0 items` → 切递归型

或用「🔍 检测」功能自动判断（需有效 session）。

## 4. 技术选型理由

| 选择 | 理由 |
|------|------|
| SQLite | 单机部署，无需额外数据库进程，数据量小(<100MB) |
| Flask | 轻量，满足 Web 面板需求，无过度抽象 |
| Jinja2 | 服务端渲染，无需前端构建工具链 |
| requests | HTTP 客户端，支持 Session/Cookie 管理 |
| APScheduler | Python 定时任务，比 cron 更灵活 |
| cryptography | AES 加密，Python 官方推荐 |
| bcrypt | 密码哈希 |
| PyJWT | JWT 认证 |
