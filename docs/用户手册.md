## 1. 系统概述

绿盟升级监控平台用于监控 [update.nsfocus.com](https://update.nsfocus.com) 上 6 款安全产品（WAF / IPS / IDS / RSAS / NF / UTS）的升级包发布，发现新版本后通过企业微信/钉钉/飞书/邮件通知客户。

### 监控产品

| 产品 | 说明 |
|------|------|
| WAF | Web 应用防火墙 |
| IPS | 网络入侵防护系统 |
| IDS | 网络入侵检测系统 |
| RSAS | 远程安全评估系统 |
| NF | 下一代防火墙 |
| UTS | 统一威胁管理系统 |

### 通知渠道

| 渠道 | 用途 |
|------|------|
| 企业微信 | 机器人 Webhook 推送 |
| 钉钉 | 机器人 Webhook 推送 |
| 飞书 | 机器人 Webhook 推送 |
| 邮件 | SMTP 邮件（支持附件） |

### 采集模式

| 模式 | 频率 | 耗时 | 说明 |
|------|------|------|------|
| Quick 扫描 | 每4小时（可配置） | ~3-5分钟 | 增量扫描，检测页面变化 |

## 2. 快速入门

### 第一步：添加 Session

1. 登录 [update.nsfocus.com](https://update.nsfocus.com)
2. 按 F12 → Application → Cookies → 复制 `PHPSESSID` 的值
3. 在平台「Session 管理」粘贴并验证

### 第二步：添加通知渠道

在「通知渠道」页面添加至少一个渠道：
- **企业微信/钉钉/飞书**：填写机器人的 Webhook URL
- **邮件**：填写 SMTP 服务器信息、账号密码、收件人

### 第三步：添加客户

在「客户管理」页面添加需要接收通知的客户，填写名称、公司、邮箱等。

### 第四步：创建订阅规则

在「订阅规则」页面创建规则：
1. 选择关联客户
2. 设置订阅条件（产品 + 版本 + 包类型）
3. 选择通知渠道
4. 保存

创建后系统自动开始监控，发现新包即时推送。

## 3. 仪表盘

系统首页，展示整体运行状态。

### 统计卡片

- **活跃 Session**：当前可用的 Cookie 数量
- **监控快照**：数据库中活跃的升级包快照总数
- **调度器**：当前状态（就绪 / 采集中）
- **今日推送**：今日推送总数 / 成功 / 失败
- **本周推送**：本周累计推送统计

### 采集状态

- **上次采集**：最近一次采集完成的时间（本地时间）
- **下次采集**：预计下次采集时间
- **手动采集按钮**：可手动触发 Quick 或 Full 扫描

### 最近推送

显示最近 N 天的推送明细，可按 1天/7天/30天/1年 筛选。

### 产品卡片

按产品分组显示最新的升级包概览。

### 包发布趋势图（左）

Canvas 多线折线图，展示近7/30/90天各产品每日发布包数量趋势。

- **图例切换**：点击产品标签可显示/隐藏对应曲线（斜体+删除线为隐藏）
- **悬停提示**：鼠标在图表上移动显示垂直参考线+当天各产品数值气泡
- **时间范围**：通过图表下方或右侧的 7天/30天/90天 按钮切换

### 产品发布分布（右）

表格展示近N天各产品发布包数量及占比。

- 按数量降序排列
- 每行显示：产品名称、包子、占比进度条
- 通过右侧 7天/30天/90天 按钮切换时间范围

绿盟站点需要登录才能访问升级包详情。PHPSESSID 有过期时间（约24分钟无操作），系统每30分钟自动发送心跳保活。

### 操作

| 操作 | 说明 |
|------|------|
| 添加 | 粘贴 PHPSESSID，系统自动验证有效性 |
| 验证 | 手动验证 Session 是否有效 |
| 删除 | 删除失效的 Session |
| 心跳历史 | 查看 Session 心跳记录（成功/失败/延迟） |

## 5. 通知渠道

### 添加渠道

**IM 渠道（企微/钉钉/飞书）**：
- Webhook URL：机器人的完整 Webhook 地址
- Secret（可选）：签名密钥

**邮件渠道**：
- SMTP 服务器地址和端口
- 用户名和密码（授权码，非登录密码）
- 发件人名称
- 收件人列表（逗号分隔）
- **每小时/每天发送上限**（0=不限制，可选配置，防止单渠道被异常流量打满）

### 邮件频率限制

为防止 Bug 导致邮件轰炸，系统对所有邮件发送实施三层限流：

| 层级 | 配置位置 | 默认值 | 超限行为 |
|------|---------|--------|---------|
| 渠道级 | 渠道编辑页（每个SMTP独立） | 0=不限制 | 跳过该渠道，规则内其他渠道仍可用 |
| 客户级 | 系统配置页 | 10封/时，50封/天 | 跳过该客户本次通知 |
| 全局级 | 系统配置页 | 100封/时，500封/天 | 停止所有邮件发送 |

> 超限时写 WARNING 日志，DeliveryResult 标记为失败并记录原因。计数器 DB 持久化，服务重启不丢。

### 安全说明

编辑已有渠道时，Webhook URL 中的 `key` 参数会被掩码显示（`key=********`），SMTP 密码和 Secret 不显示。留空不修改则保留原值。

### 测试

每个渠道都有「测试」按钮，可发送一条测试消息验证连通性。

### 支持邮箱

| 邮箱 | SMTP | 端口 | 说明 |
|------|------|------|------|
| QQ邮箱 | smtp.qq.com | 465 | 需开启SMTP服务，使用授权码 |
| 163邮箱 | smtp.163.com | 465 | 需开启SMTP服务 |
| 126邮箱 | smtp.126.com | 465 | 需开启SMTP服务 |

## 7. 订阅规则

订阅规则是系统的核心——决定**什么产品的新包**通过**哪些渠道**通知**哪个客户**。

### 规则列表

| 列 | 说明 |
|----|------|
| 📢 | 回退通知开关图标 |
| 名称 | 规则名称 |
| 客户 / 收件人 | 关联客户名称及邮箱 |
| 通知对象 | 实际接收通知的目标，优先取规则级客户邮箱，机器人渠道则显示「名称(类型)」 |
| 订阅条件 | 订阅的产品/版本/类型条件，「📡 全部产品」表示监控全部 |
| 推送 | 即时 / 每周 / 每月 / 每季 |
| 有效期 | 规则有效期 |
| 状态 | 启用/停用（点击切换） |

### 创建/编辑规则

| 字段 | 说明 | 必填 |
|------|------|------|
| 规则名称 | 便于识别的名称 | ✅ |
| 📡 订阅全部产品与类型 | 勾选后不设筛选条件，监控所有产品的全部升级包 | |
| 关联客户 | 从客户列表选择 | ✅ |
| 客户邮箱 | 覆盖客户档案邮箱，多个逗号分隔 | |
| 订阅条件 | 产品+版本+类型，可添加多组（勾选「全部」后隐藏） | |
| 延迟(天) | 发现后延迟N天推送，0=立即推送 | |
| 推送模式 | 即时/每日汇总/每周汇总/每月汇总 | |
| 有效期至 | 规则失效日期 | |
| 通知渠道 | 可添加多个渠道 | |
| 附件大小(MB) | 0=使用全局设置 | |
| 回退通知 | 是否通知包撤回 | |

### 📡 订阅全部产品与类型

勾选此选项后：
- 订阅条件选择区自动隐藏
- 规则会匹配**所有 6 个产品**的**全部类型**升级包
- 列表显示蓝色「📡 全部产品」标识
- 相当于 `filter_conditions = {}`，后端空条件默认匹配全部

### 删除保护

- 删除客户前检查是否被订阅规则引用，有引用则拒绝并提示规则名
- 删除渠道前同样检查引用关系
- 需先在订阅规则中取消关联后再删除

### 通知渠道说明

> **重要**：系统只通过规则中勾选的渠道发送通知。关联客户有邮箱 ≠ 自动发邮件，必须显式勾选邮件渠道才会发送邮件。

### 延迟推送策略

| 策略 | 行为 |
|------|------|
| 重置 | 每次新包重新计时，延迟窗口内只发一次 |
| 追加 | 延迟窗口内新包追加到同一批次 |
| 窗口 | 固定时间窗口推送 |

### 汇总模式

| 模式 | 行为 |
|------|------|
| 即时 | 新包立即推送 |
| 每周 | 每周指定日期合并推送一次 |
| 每月 | 每月指定日期合并推送一次 |
| 每季 | 每季度合并推送一次 |

## 8. 数据采集

展示当前监控的所有升级包快照。

### 页面功能

| 功能 | 说明 |
|------|------|
| 产品分组 | 按产品分组，可折叠/展开 |
| 版本分组 | 按版本分支分组 |
| 包详情 | 点击行展开查看完整信息（含页面HASH：旧hash→新hash） |
| **推送按钮** | 手动推送某个包（按客户/渠道/手动邮箱） |
| **推送频率限制** | 同邮箱/渠道 1分钟最多5次，超限10分钟禁用 |

### 推送模态框

点击包行的「推送」按钮，三种模式：

1. **👤 按客户**（默认）：选择有邮箱的客户，系统自动通过邮件渠道中继发送到客户邮箱
2. **📡 按通知渠道**：选择单个渠道直接推送（email渠道使用其配置的收件人）
3. **✉️ 手动填写邮箱**：直接输入邮箱地址，多个用逗号分隔，实时格式校验

> **注意**：手动推送忽略客户订阅的产品/版本限制，直接发送当前快照内容。

### 推送频率限制

防止恶意利用手动推送功能：

| 规则 | 说明 |
|------|------|
| 频率上限 | 同一邮箱或渠道 1 分钟内最多 5 次推送 |
| 超限处罚 | 超过后该邮箱/渠道推送功能禁用 10 分钟 |
| 恢复方式 | 10 分钟后自动恢复，或管理员通过 API 手动重置 |
| 重置 API | `POST /api/system/rate-limits/reset` `{"key":"email@x.com"}` 或 `{}` 全部 |

超限时前端按钮显示倒计时「已禁用 M:SS」，到时自动恢复可点击。

## 10. 系统配置

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **🔧 维护模式** | 关闭 | 开启后所有推送静默，采集照常 |
| 采集间隔 | 4 小时 | Quick 扫描间隔 |
| 回退确认次数 | 2 | 连续N次全量扫描缺失才确认撤回 |
| 速率下限 | 1.0 秒 | HTTP 请求最小间隔 |
| 速率上限 | 3.0 秒 | HTTP 请求最大间隔（随机） |
| 附件上限 | 10 MB | 邮件附件最大大小（全局） |
| 快照保留 | 180 天 | 超过后自动清理 |
| 静默开始 | 22:00 | 静默时段开始 |
| 静默结束 | 08:00 | 静默时段结束 |
| 采集间隔 | 4 小时（可配置） | Quick 扫描间隔 |
| 心跳间隔 | 30 分钟 | Session 保活间隔 |

### 📧 邮件频率限制

防止 Bug 导致邮件轰炸的硬上限配置：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 全局-每小时上限 | 100 | 整个系统每小时最多发N封，0=不限制 |
| 全局-每天上限 | 500 | 整个系统每天最多发N封 |
| 客户-每小时上限 | 10 | 每个客户独立计数，每小时最多N封 |
| 客户-每天上限 | 50 | 每个客户独立计数，每天最多N封 |

> 渠道级别的限制在「通知渠道」编辑页单独配置，优先级：渠道 > 客户 > 全局

### 维护模式

开启后：
- 采集照常运行
- 所有推送静默（即时/延时/手动/摘要）
- 适合调试和测试环境

### 包分类配置

可配置哪些包类型属于「全量包」（离线升级包），用于摘要汇总时去重。

## 12. 常见问题

### Q: 为什么仪表盘时间比实际少8小时？
A: 已修复。系统使用 UTC 存储，前端自动转为浏览器本地时间（中国=UTC+8）。

### Q: 为什么 Quick 扫描没有发现新包？
A: Quick 扫描需要先运行一次来填充 `source_url`。首次部署会自动触发。之后按配置的间隔检测已知页面变化。

### Q: 邮件发送失败？
A: 检查：
1. QQ邮箱需用授权码非登录密码
2. 云服务器 IP 可能被网易邮箱风控，建议用 QQ 邮箱
3. 端口 465 使用 SSL，25/587 使用 STARTTLS

### Q: 如何不让某客户收到撤回通知？
A: 编辑该规则，取消勾选「📢 包回退通知」。

### Q: 如何临时关闭所有推送？
A: 系统配置 → 开启「🔧 维护模式」。

### Q: 推送被频率限制了怎么恢复？
A: 两种方式：
1. 等待 10 分钟自动恢复
2. 管理员调用 API 立即重置：
```bash
# 重置某个邮箱
curl -u admin:<your_password> -X POST http://127.0.0.1:9999/api/system/rate-limits/reset \
  -H 'Content-Type: application/json' -d '{"key":"user@example.com"}'
# 重置全部
curl -u admin:<your_password> -X POST http://127.0.0.1:9999/api/system/rate-limits/reset \
  -H 'Content-Type: application/json' -d '{}'
```
A: 使用天数筛选（默认30天），或点击「清空」按钮清理旧记录。

### Q: Channel 密码会被泄露吗？
A: 编辑渠道时 webhook key 显示为 `********`，密码不显示。留空不修改保留原值。
