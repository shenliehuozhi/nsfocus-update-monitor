# 需求规格说明

> 项目：绿盟升级监控平台 (NSFOCUS Update Monitor)  
> 版本：v2.0  
> 日期：2026-05-25  
> 前置文档：需求说明.md → 系统架构.md → 数据模型.md

---

## 1. 项目背景

绿盟科技通过升级站点 update.nsfocus.com 定期发布安全产品升级包。目前绿盟不会主动通知客户升级包发布，客户需自行登录站点查看，导致：
- 关键安全更新可能被遗漏
- 客户无法及时了解规则库/系统升级包发布
- 运维人员需要手动逐个检查多个产品

本平台解决：自动监控 → 智能检测 → 多渠道通知 → 客户管理闭环。

## 2. 用户角色

| 角色 | 说明 |
|------|------|
| **系统管理员** | 管理平台全局配置、管理所有客户、推送通知 |
| **普通运营人员** | 管理自己的 Session、订阅规则、查看推送历史 |

一期不做角色权限细分，所有用户平权。

## 3. 功能需求

### 3.1 Session 管理

- **FR-01**: 用户可手动输入绿盟站点的 PHPSESSID ✅
- **FR-02**: 系统展示 Session 状态（有效/过期/污染/未知）✅
- **FR-03**: Session 过期前主动发送告警（企业微信通知）✅
- **FR-04**: 多用户 Session 池化，一个失效自动切换其他用户的可用 Session ✅
- **FR-05**: Session 加密存储（AES-256-GCM）✅
- **FR-06**: Session 按用途（discover/collect）和模式（standard/vm）分类管理 ✅
- **FR-07**: Session 心跳健康检查，实时监控有效性 ✅

### 3.2 内容源采集

- **FR-08**: 支持产品动态管理（自动发现 + 手动注册 + 开关控制）✅
- **FR-09**: 自动遍历多级页面结构（产品页 → 版本页 → 包列表页）✅
- **FR-10**: 解析升级包字段：文件名、版本号、MD5、大小、描述、发布时间、下载 ID ✅
- **FR-11**: 从描述中提取结构化信息：新增/修改/删除规则、前置版本、紧急程度 ✅
- **FR-12**: 请求间随机延迟 1-3 秒，防止限流 ✅
- **FR-13**: Quick 采集模式：页面 hash 对比，无变化则跳过详情抓取 ✅
- **FR-14**: 解析器健康检查（解析到的包数异常变化时告警）✅
- **FR-15**: 包类型路径自动发现（discover_package_types）✅

### 3.3 变化检测

- **FR-16**: 新增检测：本次有、上次无的包标记为新增 ✅
- **FR-17**: 回退检测：上次有、本次连续 N 次（默认 2 次）无的标记为回退 ✅
- **FR-18**: 防误报：单次缺失不判定回退，需连续确认 ✅
- **FR-19**: 回退通知不受延迟/静默限制，即时推送 ✅
- **FR-20**: 包撤回时取消 delay/digest 队列中的相关条目 ✅

### 3.4 客户管理

- **FR-21**: 客户档案：名称、公司、联系人、邮箱、持有产品列表 ✅
- **FR-22**: 客户与通知渠道关联（rule_channels.customer_id）✅
- **FR-23**: 客户删除前检查是否被订阅规则引用 ✅

### 3.5 订阅规则

- **FR-24**: 规则包含：匹配条件（链路径/关键词/紧急程度）、通知渠道、推送策略 ✅
- **FR-25**: 推送模式：即时推送（delay 天延迟）和汇总推送（定时窗口）✅
- **FR-26**: 即时推送延迟：可设置 0~30 天观察期 ✅
- **FR-27**: 窗口策略：配置周几 + 时间段，在窗口内合并推送（汇总模式）✅
- **FR-28**: 静默时段：可配置不推送的时间段（如 22:00~08:00）✅
- **FR-29**: 回退通知不受延迟/静默限制，即时推送 ✅
- **FR-30**: 订阅条件支持链路径匹配（精确到产品→子分类→版本→包类型）✅
  - leaf 模式：精确匹配具体包类型
  - subtree 模式：订阅某节点下全部
- **FR-31**: 规则过期（valid_until）不触发推送 ✅

### 3.6 通知渠道

- **FR-32**: 企业微信机器人（Webhook + Markdown）✅
- **FR-33**: 钉钉机器人（Webhook + Markdown）✅
- **FR-34**: 飞书机器人（Webhook + Markdown）✅
- **FR-35**: 邮件（SMTP，支持附件）✅
- **FR-36**: 邮件附件策略：规则包 ≤10MB 自动下载并附件发送；系统包 >10MB 仅提供链接 ✅
- **FR-37**: 通知模板支持变量：产品、版本、包类型、文件名、MD5、大小、发布时间、新增/修改规则摘要、依赖版本、下载链接、紧急程度 ✅

### 3.7 推送确认

- **FR-38**: 每次推送完成后，向对应 IM 群发送送达确认消息 ✅

### 3.8 推送历史

- **FR-39**: 记录所有推送历史，支持按时间/产品/客户筛选 ✅
- **FR-40**: 支持从历史记录一键重新推送 ✅
- **FR-41**: 展示推送统计：本月推送次数、各产品发布频次 ✅

### 3.9 Web 管理面板

- **FR-42**: 用户注册/登录（密码 bcrypt 哈希）✅
- **FR-43**: 首次使用引导流程：① 获取 Session → ② 添加渠道 → ③ 创建规则 ✅
- **FR-44**: 内容预览：配置监控项后可手动触发一次抓取预览结果 ✅
- **FR-45**: 手动推送：按客户/渠道/手动邮箱三种模式推送快照 ✅
- **FR-46**: 监控仪表盘：Session 状态、最近抓取时间、各产品包数、推送统计 ✅

### 3.10 系统事件通知

- **FR-47**: 采集汇总通知（有声/有错/空跑三态）✅
- **FR-48**: Session 异常通知（区分具体用户异常 vs 系统健康检查告警）✅
- **FR-49**: 系统事件通知可独立配置开关和通知渠道 ✅

### 3.11 安全

- **FR-50**: 敏感字段 AES 加密存储（Session、Webhook URL、SMTP 密码）✅
- **FR-51**: Web 面板 JWT 认证 + 登录限流（5次/15分钟封禁）✅
- **FR-52**: CORS 配置通过环境变量控制 ✅
- **FR-53**: 操作审计日志：登录、配置变更、手动推送 ✅
- **FR-54**: 所有 API 加 @require_auth 装饰器 ✅

### 3.12 运维

- **FR-55**: 结构化日志（DEBUG/INFO/WARN/ERROR）✅
- **FR-56**: 调度器全局开关（scheduler_enabled）✅
- **FR-57**: SQLite WAL 模式 + busy_timeout 配置 ✅
- **FR-58**: 采集异常实时检测（关键字段空/报错）✅

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 全量检查 6 个产品 ≤ 3 分钟 |
| 内存 | 运行时 ≤ 300MB（含 Flask + SQLite） |
| 可靠性 | 采集失败自动重试 3 次 |
| 可扩展 | 新增产品只需添加配置文件，无需修改核心逻辑 |
| 数据保留 | 快照数据保留 180 天，推送历史保留 365 天 |
